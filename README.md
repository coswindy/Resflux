# Resflux

Resflux 是一个面向中文内容站、科技博客和资源整理类网站的 WordPress 主题。主题在常规博客展示的基础上，补充了首页模块管理、SEO 基础设置、广告位管理、资源聚合、网址导航、投稿表单和若干安全优化选项。

项目以实用性和可维护性为主要目标，功能设计偏向轻量集成，尽量减少对额外插件的依赖。

## 主要功能

- 响应式博客布局，支持首页网格与列表两种文章展示方式。
- 首页模块可配置，包含精选轮播、轮播下方广告、分类文章流、资源聚合、网址导航、最新文章和首页侧栏。
- 文章页支持阅读进度、阅读时间、浏览量统计、相关文章、作者信息、上一篇/下一篇导航和社交分享。
- 支持深色模式、返回顶部、移动端菜单、搜索弹层和长文折叠。
- 内置 SEO 基础设置，包括首页标题、描述、关键词、Open Graph、canonical、自动内链、图片重命名和 XML 站点地图。
- 内置广告管理，支持 Google AdSense 自动广告和多个手动广告位。
- 提供资源聚合与网址导航两类后台内容类型，适合整理工具、站点、文档或外部链接。
- 提供前台登录、注册和投稿表单短代码，投稿内容默认进入待审核状态。
- 支持页脚版权、备案信息、二维码、社交链接、Header/Footer 自定义代码。
- 提供基础安全选项，如隐藏 WordPress 版本信息、禁用 XML-RPC、移除部分发现链接、禁用 Emoji 资源、登录错误信息模糊化等。
- 内置常用小工具：热门文章、最新文章、最新评论、分类目录和广告位。

## 环境要求

- WordPress 5.9 或更高版本建议使用。
- PHP 7.4 或更高版本。
- 主题不依赖构建流程，上传后即可在 WordPress 后台启用。

## 安装方式

1. 将主题目录放入 WordPress 的 `wp-content/themes/` 目录。
2. 确认目录名为 `Resflux` 或其他你希望使用的主题目录名。
3. 登录 WordPress 后台，进入「外观」->「主题」，启用 Resflux。
4. 进入「外观」->「主题设置」，根据站点需求配置 SEO、外观、首页模块、广告和安全选项。

## 常用配置

启用主题后，建议优先检查以下项目：

- 「外观」->「菜单」：配置主导航和页脚导航。
- 「外观」->「主题设置」->「外观设置」：设置配色、Logo、首页布局、文章页布局和长文折叠高度。
- 「外观」->「主题设置」->「首页模块」：选择首页启用模块，并调整模块顺序。
- 「外观」->「主题设置」->「SEO 设置」：配置首页 SEO 信息、Open Graph 图片和站点地图。
- 「外观」->「主题设置」->「广告设置」：填写 AdSense Publisher ID 或自定义广告代码。
- 「外观」->「主题设置」->「页脚设置」：填写版权、备案信息和二维码。

## 首页模块

首页模块通过主题设置控制，可用模块包括：

- `featured`：精选轮播。
- `carousel_ad`：轮播下方广告。
- `category_feed`：分类文章流。
- `resources`：资源聚合。
- `navigation`：网址导航。
- `latest`：最新文章。
- `sidebar`：首页侧栏。

模块顺序使用英文逗号分隔。未启用的模块会自动跳过。

## 短代码

主题提供两个前台短代码：

```text
[techpress_login_form]
[techpress_submit_form]
```

`[techpress_login_form]` 用于输出登录和注册表单。  
`[techpress_submit_form]` 用于输出投稿表单，提交后的文章状态为待审核。

如果使用默认页面模板，主题也提供 `page-login.php` 和 `page-submit.php` 两个页面模板。

## 内容类型

主题会注册以下后台内容类型：

- `tech_resource`：资源聚合内容，可填写资源地址、资源类型和简短描述。
- `tech_navigation`：网址导航内容，可填写站点地址、Logo 地址和站点描述。

这些内容类型默认不公开独立访问，主要用于首页模块展示。

## 广告位

主题内置以下广告位：

- 轮播下方横幅。
- 首页网格信息流。
- 文章顶部广告。
- 文章中间广告。
- 文章底部广告。
- 列表页信息流。
- 页脚横幅。
- 侧栏广告小工具。

广告代码会由后台设置保存并在对应位置输出。使用第三方广告代码前，请确认代码来源可信，并符合站点所在地的合规要求。

## 注意事项

- 前台投稿功能依赖 WordPress 用户权限。默认情况下，只有具备 `edit_posts` 能力的用户可以提交文章。
- SEO、站点地图和安全设置适合基础使用；如果站点已有专业 SEO 或安全插件，建议避免重复开启相同功能。
- Header/Footer 自定义代码会直接输出到页面，请仅添加可信代码。
- 在生产站点启用前，建议先备份数据库，并在测试环境确认菜单、固定链接、缓存插件和广告脚本表现正常。

## 开发与检查

当前代码为传统 WordPress 主题结构，不需要前端构建步骤。常用检查命令：

```bash
find . -type f -name '*.php' -print0 | xargs -0 -n1 php -l
find assets -type f -name '*.js' -print0 | xargs -0 -n1 node --check
```
